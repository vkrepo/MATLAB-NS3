Truck Platooning

Introduction

Truck platooning enables a group of trucks to travel closely together, maintaining a safety gap. The platoon leader governs the movement of following trucks. This is achieved through V2V communication.

Aim of the example

The example models a vehicular communication network involving a truck-platoon, an RSU (road-side unit), and a few non-platoon vehicles in adjacent lanes. It analyzes the performance of truck platoon by varying the parameters at various layers of the communications system.

MATLAB-NS3 Interface

The core simulation runs in NS-3 with scenario description written in MATLAB. The NS-3 library functions are called via Mex-bindings interface from MATLAB. Along with the scenario, the features implemented in MATLAB are:
  • Highway Topology creation.
  • Packet construction of all types.
  • Mobility intelligence based on the received packets.
  • Post-simulation visualization.

What is not in scope of this demo

  • Lateral movement i.e. no steering involved. No backward movement either.
  • Dynamic platoon formation (or segregation)

Scenario Description

A pre-formed platoon of trucks moves on a lane of a straight highway. Other lanes of the highway have non-platoon traffic.
All the trucks have WAVE stack (over 802.11p) installed on them. As only straight-line motion is covered, the only controls enabled on vehicles are acceleration and deceleration. The aim is to ensure that gap between consecutive trucks is within an acceptable range so that they are neither too close nor too far apart, using the V2V platoon-beacon packet transmissions. The lower and upper limit of acceptable gap can be configured by the user.
If platoon leader decelerates, the following trucks decelerate to maintain safe gap and avoid rear-end collision. Likewise, the trucks accelerate if their respective preceding truck accelerates, to make sure that trucks are within a threshold distance of each other.
To simulate deceleration of platoon trucks, a rough patch of road with a lower speed limit is created. An RSU which periodically broadcasts warning packets, is placed near this rough patch. When platoon leader listens to the warning packet, it decelerates (only if required as platoon might already be moving at lesser pace) such that velocity reduces to acceptable speed limit of rough patch. Following the platoon leader, based on V2V communication, other trucks on the platoon decelerate too.
Once platoon leader crosses the rough patch it starts accelerating, which in turn makes other following trucks accelerate too for limiting the inter-vehicular gap.
Note: The platoon leader starts accelerating as soon as it crosses the rough patch and broadcasts the same. So, the trailing trucks (which are unaware that they are on rough patch) might start accelerating even before they cross the rough patch.

Communication model

Scenario involves three different packet types, each sent periodically:
  • Platoon-beacon transmitted by trucks in the platoon.
a) With information of sender: instantaneous position, velocity and acceleration.
b) Only processed by the truck just behind the sending one for acceleration/deceleration decisions.
  • Rough patch warning transmitted by Road-Side Unit.
a) With information about rough patch of road: start point, length and speed limit on it.
b) Only processed by platoon leader for decelerating.
  • Packets sent by non-platoon vehicles, to model network interference
Quite expectedly, communication loss can hamper the platoon-movement. If a truck loses a platoon-beacon from its preceding truck due to network conditions, it does not get its updated position/ velocity/ acceleration information. The affect gets amplified if it loses a few consecutive packets at a stretch, affecting the stability or safety of the platoon. So, the platoon is said to be failed if burst-loss is more than a bearable value.
Similarly, if platoon leader receives rough patch warning only when it is very near to rough patch, it might have to decelerate aggressively (which can be uncomfortable) or might even go over the rough patch with speed over the safety limit.
NOTE: All the packets of vehicles (platoon and non-platoon) are sent and received on CCH channel as WSMP (WAVE Short Message Protocol) packets.

Configuration Parameters

a) Communication
  • PHY parameters for platoon and non-platoon trucks
i) TX gain, Rx gain, Rx Noise figure
  • Log-distance propagation loss model parameters
ii) Path loss exponent, reference distance, reference loss.
  • Application level parameters
i) Platoon-beacon generation frequency.
ii) Rough Patch warning packet generation frequency.
iii) Non-platoon vehicle packet generation frequency.
  • Limit on consecutive platoon-beacon loss, to consider it as unsafe.
b) Highway Model and Other vehicles
  • Highway Length, Lane count, Lane width
  • Rough road patch (start & length), safe speed limit on rough road patch.
  • Start positions for the non-platoon vehicles on their respective lanes.
  • Velocity of the non-platoon vehicles
c) Platoon vehicles configuration
  • Number of trucks in the platoon, and truck length
  • Initial standstill gap between trucks in the platoon.
  • Max speed limit, Max deceleration/acceleration allowed.
  • Min/Max allowed gap between consecutive trucks (in terms of time units)
d) Mobility Model for vehicles - Constant Acceleration Model & its parameters

Results and Analysis

While the simulation is running, the results are gathered into log files. The results include:
a) Packet statistics at application level for each truck in the platoon
  • Number of platoon-beacons transmitted by each truck
  • Number of preceding trucks platoon-beacons received successfully (0 for 1st truck)
  • Number of preceding trucks platoon-beacons corrupted
  • Worst gap between reception of platoon-beacon from preceding truck
  • Burst-losses (i.e. consecutive lost packets) count for platoon-beacons from preceding truck
  • Total number of packets received successfully, including those from non-platoon vehicles
b) MAC/PHY statistics for each truck in the platoon
  • Total number of corrupted Rx packets, including those from non-platoon vehicles
A visualization script reads the simulation results from log files, analyzes them and performs visualization. The visualization includes:
  • Graphical visualization of highway, platoon, other vehicles and their movement.
  • Instantaneous velocity, acceleration and gap for each truck in the platoon
  • Packet metrics
  • In case the vehicles are closer than the safe gap, the corresponding vehicle changes to red color.
  • If the platoon communication is lost for a truck (reached the limit of consecutive platoon-beacon losses), a message pops up in the visualizer indicating platoon-failure.
